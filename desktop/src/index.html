CSP

启用/停用的改动会在下次点击「启动 Claude Science」时生效；若沙箱正在运行，会自动重启以应用。

还没有 Skill。

点「新建」从零创建，或在「⋯」里选「导入」一个含 SKILL.md 的目录。

新增/编辑/启用的改动会在下次点击「启动 Claude Science」时生效；若沙箱正在运行，会自动重启以应用。

还没有本地 MCP。

点「新增」，配置一个本地 stdio 连接器（command + args + env）。启用后会在启动沙箱时写入 <data-dir>/mcp/local-mcp.json。

扫描本地 MCP

扫描常见 AI 客户端的 MCP 配置：Cursor、Claude Desktop、Claude Code、Codex（~/.codex/config.toml）、Devin Desktop（原 Windsurf）、VS Code（~/Library/Application Support/Code/User/mcp.json）、Zed，以及国产 Qoder / 通义灵码（阿里）、Trae / TRAE SOLO（字节）、CodeBuddy（腾讯）。识别 mcpServers / servers / context_servers 等键，勾选后导入为 CSP 本地 stdio MCP。仅识别本地 stdio（带 command）连接器；环境变量只显示 key，导入时由后端复制原值。

没有扫描到可导入的本地 stdio MCP。

新增本地 MCP

仅限字母、数字、- _ .，作为连接器标识。

运行时命令（python3 / node / npx …）或绝对路径。

引用的绝对路径会自动加入沙箱读权限（user_read_paths）。

编辑时：KEY= 留空保留原值，删除整行则删除该变量，填新值则覆盖。密钥不会回显。

扫描本地 Skill

扫描 ~/.agents/skills、~/.codex/skills、~/.claude/skills、~/.cursor/skills，以及国产 ~/.trae/skills / ~/.trae-cn/skills（字节）、~/.codebuddy/skills（腾讯），勾选后复制到 ~/.csp/skills/。沙箱自带的科学 Skill 不在此列。

没有扫描到可导入的 Skill。

从 Science 工作区采纳

扫描沙盒内 workspaces/ 下的 Skill 草稿（*.skill.md、*_SKILL.md 或含 SKILL.md 的目录），连同 kernel.py 等伴随文件复制到 ~/.csp/skills/。Science 无法直接发布技能时请用此功能。

没有扫描到可采纳的 Skill 草稿。

新建 Skill

作为 Skill 名称写入 SKILL.md 的 frontmatter。同名 Skill 不可重复。

编辑上方名称/描述会自动同步到 frontmatter；手动改动正文后以此处内容为准。保存后写入 ~/.csp/skills/。

导入 Skill

选择包含 SKILL.md 的目录，将被复制到 ~/.csp/skills/。
